# Evidence Bundle Layout

The evidence bundle writes one customer-scoped directory per assembled bundle:

```text
<data-root>/artifacts/<customer>/evidence-bundle/<change_id>-<timestamp>/
├── bundle.json
├── manifest.json
├── evidence.md
├── evidence.md.sig
└── signature.json
```

- `bundle.json` is the canonical merged payload used for validation and rendering.
- `manifest.json` is a normalized copy of the supplemental manifest.
- `evidence.md` is rendered from the active compliance adapter template.
- `evidence.md.sig` is the detached signature generated by the configured signing backend.
- `signature.json` records the signing method, key reference, identity, and signature path.
