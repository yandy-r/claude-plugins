# Go Testing Command

Provide Go testing patterns and TDD guidance using idiomatic `testing` package practices.

**Load and follow the `ycc:go-testing` skill**, passing through `$ARGUMENTS`.

The skill covers:

- Table-driven tests with `t.Run` subtests
- `t.Helper`, `t.Cleanup`, and `t.TempDir` lifecycle helpers
- Benchmarks with `testing.B` and Go 1.18+ fuzzing (`f.Fuzz`)
- HTTP handler testing with `httptest`
- Golden files and interface-based mocks
- Test coverage and TDD workflow
