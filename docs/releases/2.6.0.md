# Release 2.6.0 — 2026-04-20

## Summary

Adds a `--worktree` flag to `/ycc:code-review` and `/ycc:review-fix` so concurrent reviews on different PRs run in isolated git worktrees keyed by PR number and (for fixes) by severity — no more branch collisions. Also ships the `feat(init)` copilot-instructions + pr-title template pair and a grouped agent list in the inventory README.

## Upgrade Notes

No action required — all changes are additive and backward-compatible:

- `/ycc:code-review` and `/ycc:review-fix` work identically without the new flag. Pass `--worktree` opt-in when you want isolation for concurrent reviews.
- `setup-worktree.sh parent` gains an optional `--base-ref <branch>` flag; existing two-arg callers (`parent <repo> <slug>`) behave exactly as before.
- Review-fix auto-detects a `## Worktree Setup` section in the review artifact, so passing `--worktree` on `/ycc:code-review` automatically propagates isolation into the fix phase without any flag on `/ycc:review-fix`.
