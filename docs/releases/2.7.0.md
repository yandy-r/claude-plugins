## Upgrade Notes

No breaking changes. To pick up the refreshed user-global ruleset (including
the new "default to worktrees" SHOULD), re-run:

```bash
./install.sh --target all --settings
```

opencode users specifically: `~/.config/opencode/AGENTS.md` previously mirrored
this repo's contributor rules; after re-running `install.sh --target opencode
--settings`, it will point at the generic user-global ruleset from
`ycc/settings/rules/CLAUDE.md`. The contributor-specific guidance never
belonged in a user-global file — the old behavior was a wiring mistake.

`/ycc:init` scaffolds now emit a two-pass `pr-title-autofix.yml` that both
strips placeholder prefixes (`[WIP]`, `Draft:`, `Initial plan`) and normalizes
leading Conventional Commit type words (`Refactor X` → `refactor: X`). Existing
projects: re-run `/ycc:init --templates --update --force` if you want the new
workflow.
