# Release 3.1.1 ‚Äî 2026-04-23

## Summary

Install-layer patch release. The `ycc` bundle content is effectively
unchanged since v3.1.0 (one documentation touch in
`ycc/skills/_shared/references/target-capability-matrix.md` covering
the Claude base step added in v3.1.0). Almost all work in this window
is at the install-harness layer: `install.sh` picks up a `--mode
{local|repo}` selector and splits the old `--settings` flag into a
copy-based `--settings` and a symlink-based `--rules`, and
`package.json` grows matching aliases plus `sync` / `validate` families
for bundle regeneration without install.

## Added

- `install.sh --mode {local|repo}` ‚Äî choose whether the marketplace source
  is the local repo checkout or the upstream github repo
  `yandy-r/claude-plugins@main` (claude + codex targets; cursor/opencode
  reject `--mode repo` since they have no remote-source concept).
- `install.sh --rules` ‚Äî symlink `CLAUDE.md` / `AGENTS.md` (and
  codex's `default.rules`) so shared rules keep flowing across systems.
  Uses the existing `link_rules_file` semantics: idempotent, refuses to
  replace a user-authored rules file without `--force`.
- `install.sh` claude `base` step ‚Äî registers `ycc@ycc` as a local
  marketplace via the `claude` CLI, materializes
  `~/.claude/settings.json` as a real file so the CLI write doesn't
  pollute the committed source (commit 46b9519).
- npm aliases: per-target `install:<target>:rules`, plus `sync`,
  `sync:inventory`, `sync:cursor`, `sync:codex`, `sync:opencode`,
  `validate`, `validate:inventory`, `validate:cursor`,
  `validate:codex`, `validate:opencode`, `validate:json` ‚Äî bundle
  regeneration and validation without running `install`.

## Changed

- `install.sh --settings` now **copies** per-machine config files
  (`settings.json`, `statusline-command.sh`, `config.toml`,
  `opencode.json`) instead of symlinking them. Local edits (model,
  `effortLevel`, marketplace entries, trusted-project lists, MCP
  tokens) stop back-propagating into the repo.
- `install.sh --settings` transport rules: destination missing ‚Üí copy;
  destination is a symlink ‚Üí warn and replace with copy (no `--force`
  needed, so upgrading from the old symlink flow is a no-op);
  destination is a real file ‚Üí refuse unless `--force`.
- Full-install npm aliases (`install:all`, `install:claude`,
  `install:cursor`, `install:codex`, `install:opencode`, plus their
  `:local` / `:repo` variants) now pass `--rules` alongside
  `--settings` so a single `npm run install:‚Ä¶` still installs configs
  and rules in one shot.
- `ycc/skills/_shared/references/target-capability-matrix.md` ‚Äî minor
  docs update covering the Claude base step (propagated into the
  Cursor, Codex, and opencode bundles).

## Removed

- `install.sh --target cursor --settings` and `--target cursor --only
settings` are rejected ‚Äî cursor has no per-machine config file to
  copy. Use `--rules` for the CLAUDE.md / AGENTS.md symlinks.
- `npm run install:cursor:settings` ‚Äî removed in favor of
  `npm run install:cursor:rules`.

## Upgrade Notes

This release only affects the install harness (`install.sh`,
`package.json`). The `ycc` bundle itself picked up a small docs
update ‚Äî `/reload-plugins` is sufficient to pick up the new version
once `claude plugin install ycc@ycc --scope user` has pulled it.

**Breaking (install.sh only)**:

- `install.sh --target cursor --settings` and `--only settings` for
  cursor now error out. Switch to `--target cursor --rules` (or
  `--only rules`) if you were relying on the old flag to symlink
  `CLAUDE.md` / `AGENTS.md`.
- `npm run install:cursor:settings` is removed. Use `npm run
install:cursor:rules` instead.

**Non-breaking upgrade path (all other targets)**:

On first run of the new `install.sh --settings`, each destination that
was a symlink from the old flow is replaced with a copy (with an info
line per file). No `--force` is required and no manual action is
needed. After the upgrade, per-machine edits to `~/.claude/settings.json`,
`~/.codex/config.toml`, `~/.config/opencode/opencode.json`, and
`~/.claude/statusline-command.sh` stay local and no longer appear as
modifications to the repo.

The `--rules` flag preserves the previous symlink-based flow for
`CLAUDE.md` / `AGENTS.md` (and codex `default.rules`) ‚Äî nothing about
rules file propagation has changed.
