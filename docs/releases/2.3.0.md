# Release 2.3.0 ‚Äî 2026-04-18

## Summary

Adds the `ycc:formatters` skill ‚Äî a one-command bootstrap for a best-practices lint/format environment in any target project. Also threads `--formatters` into `ycc:init`, fixes cross-target generator rewrites for `.tmpl` files, and wires a pre-push bundle-parity hook so generator drift is caught locally before CI.

## Added

- **`ycc:formatters` skill** ‚Äî profiles a project (Rust / Go / TS / Python / Docs / Shell), installs a self-contained `scripts/style.sh` bundle with tool-config templates, injects runnable aliases (package.json / Makefile / justfile), appends a "Linting & Formatting" section to the project docs, and optionally wires CI (`--ci`) and pre-commit hooks (`--hooks`). Idempotent via a `.style-bundle-manifest`; safe to re-run with `--sync`.
- **`--formatters` flag on `ycc:init`** ‚Äî delegates to `ycc:formatters` via Phase 6.5, no logic duplication.
- **`.tmpl` file support in Cursor / Codex / opencode generators** ‚Äî template files in derived bundles now receive the same target-specific rewrites (`ycc:` ‚Üí bare name, `CLAUDE.md` ‚Üí `AGENTS.md`, path prefixes) as regular markdown and shell files.

## Changed

- `scripts/generate_{cursor,codex,opencode}_skills.py` ‚Äî added `.tmpl` to `TEXT_SUFFIXES`. Propagates to several existing `init` and `hooks-workflow` template files that were previously copied byte-for-byte.
- `.pre-commit-config.yaml` ‚Äî added a local `validate-bundle` hook that runs `./scripts/validate.sh` in the `pre-push` stage. Requires a one-time `pre-commit install --hook-type pre-push` per clone.
- `CLAUDE.md` ‚Äî one-line pointer suggesting new projects bootstrap their lint/format environment via `/ycc:formatters` (or `/ycc:init --formatters`).

## Removed

- None.

## Fixed

- Derived-bundle drift when a markdown linter reflows source-of-truth tables after a `./scripts/sync.sh` run. The new pre-push hook now catches this class of drift locally; previously it was only caught in CI.
