# Release 3.0.0 — 2026-04-22

## Summary

Marketplace-shape restructure: the `yci` consulting / systems-integration plugin has been extracted into its own sibling repo, [`yandy-r/claude-infra-plugins`](https://github.com/yandy-r/claude-infra-plugins). This repository's marketplace now ships a single plugin — `ycc` — unchanged in surface, with the yci dispatch branches stripped from `scripts/sync.sh` and `scripts/validate.sh`.

## Breaking

- **`yci:{skill,command,agent}` no longer resolves from `yandy-r/claude-plugins`.** Users who were installing yci via this marketplace must switch to the sibling repo:

  ```bash
  /plugin marketplace add yandy-r/claude-infra-plugins
  /plugin install yci@yci
  ```

- `.claude-plugin/marketplace.json` now lists a single entry (`ycc`). Any tooling that relied on iterating multiple plugins in this marketplace will need to adapt.
- `scripts/validate.sh --only yci` is no longer a valid target. The validator's `VALID_TARGETS` is now `inventory, cursor, codex, opencode, json`.
- `scripts/sync.sh` no longer iterates a `PLUGINS` array; it operates directly on the `ycc` source tree.

## Added

Nothing new in the `ycc` plugin itself. All `feat(yci):*` commits in the pre-3.0.0 history now live in `yandy-r/claude-infra-plugins` (git history was preserved via `git filter-repo` during the extraction).

## Changed

- `refactor!: extract yci into sibling repo claude-infra-plugins` (e4d57a6) — removed the `yci/` source tree, `scripts/validate-yci-skills.sh`, yci-specific PRD / reports / plans under `docs/prps/`, and all yci dispatch branches from the repo-wide scripts. Rewrote `CLAUDE.md` and `CONTRIBUTING.md` to reflect single-plugin reality; retained a short "moved to" breadcrumb pointing at the sibling repo.
- `chore: enable yci@yci from sibling marketplace` (6541c49) — updated `ycc/settings/settings.json` so this repo's Claude Code session picks up yci from its new home.

## Removed

- `docs/prps/prds/yci.prd.md`, `docs/prps/reports/yci-customer-guard-report.md`, `docs/prps/plans/completed/yci-customer-guard.plan.md` (moved)

## Upgrade Notes

**For consumers of `yci`:** install from the new sibling repo and re-enable:

```bash
/plugin marketplace add yandy-r/claude-infra-plugins
/plugin install yci@yci
```

Update your `~/.claude/settings.json` `enabledPlugins` to point at the new source:

```json
"enabledPlugins": {
  "yci@yci": true,
  ...
},
"extraKnownMarketplaces": {
  "yci": {
    "source": { "source": "github", "repo": "yandy-r/claude-infra-plugins", "ref": "main" },
    "autoUpdate": true
  }
}
```

**For consumers of `ycc`:** no action required. `ycc:{skill,command,agent}` is unchanged. The sole visible difference is `/plugin marketplace update ycc` showing `3.0.0` instead of `2.7.x`.

**Why the major bump?** `ycc` itself has no surface-level changes, but the marketplace published from this repository changed shape — users who had yci enabled here will see it stop resolving. Per the version policy tie-breaker ("prefer the more conservative bump"), this is recorded as 3.0.0 rather than 2.8.0.

## Provenance

The yci → `claude-infra-plugins` split preserved full git history for every extracted path via `git filter-repo`. In the new repo, `git log -- yci/` shows the full pre-extraction trail including PRs #75 (evidence-bundle) and #76 (MOP workflow). The new repo's bootstrap commit (`13d212d chore(extract): bootstrap claude-infra-plugins from claude-plugins yci subtree`) sits on top of that preserved history and is tagged `v0.2.0-extraction`.

The 20 open yci-labeled tracking issues in `yandy-r/claude-plugins` at the time of extraction were re-filed in `yandy-r/claude-infra-plugins` (#1–#20) with provenance footers pointing back at the original claude-plugins issue numbers.
