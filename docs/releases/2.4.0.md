# Release 2.4.0 — 2026-04-18

## Summary

Lefthook-powered git hook scaffolding lands in `ycc:init` and `ycc:formatters`. Running `/ycc:init --git` now emits a working `lefthook.yml`, an idempotent cross-platform `install-lefthook.sh` bootstrap, and `docs/lefthook-usage.md` — replacing the advisory pre-commit-recommendation doc. The formatters bundle picks up `--staged` / `--unstaged` scope flags and merges into existing lefthook configs instead of clobbering. The install-skill CLAUDE.md/AGENTS.md symlinking from the v2.3.2 tail is also rolled into this release.

## Upgrade Notes

**No breaking changes for plugin consumers.** Existing `/ycc:init --git` invocations keep working and gain more output.

For contributors to this repo:

- Run `./scripts/install-lefthook.sh` after pulling to swap the local `pre-commit` framework hooks for lefthook. `pre-commit uninstall` first if your local clone still has that wired; lefthook has already backed the old hooks up to `.git/hooks/{pre-commit,pre-push}.old`.
- `npm install` now runs `lefthook install` automatically via the `prepare` script.
- New npm scripts: `lint:staged`, `lint:unstaged`, `format:staged`, `format:unstaged`.

For `ycc:init --git` users on new projects:

- The `--git` flag now emits three additional files alongside the existing `.gitmessage` (and `commitlint.config.cjs` on JS/TS projects): `lefthook.yml`, `scripts/install-lefthook.sh`, and `docs/lefthook-usage.md`. Run `bash scripts/install-lefthook.sh` in the generated project to activate the hooks. Safe to re-run; idempotent.

For `ycc:formatters --hooks` users:

- The emitted pre-commit commands now pass `--staged` to the bundled `style.sh`. When a `lefthook.yml` already exists (e.g., from a prior `ycc:init --git` run), the managed pre-commit commands are merged into it instead of overwriting — other stages (`pre-push`, `commit-msg`) are preserved. Merge requires Python 3 with PyYAML; otherwise the script degrades to a manual-merge notice.
